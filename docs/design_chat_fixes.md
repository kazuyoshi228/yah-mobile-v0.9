# 設計図 — チャット挙動の修正（dev検証で判明した4件）

対象: chat-yah-mobi ウィジェット＋AI応答
状態: **設計（実装前・承認待ち）**
経緯: dev で実機テストしたところ、顧客体験を損なう4件を確認。うち①②は前回の言語修正の過剰補正が原因。**リリース前に必須（P0）**。

---

## 問題一覧（症状 → 原因 → 修正）

### ① 回答言語がセッション言語(ja)に固定され、ユーザーの実言語/明示指定を無視 【P0】
- **症状**: 英語で話しかけても日本語で返答。"english please" にも日本語で返し、あげく**「日本語のみ対応」と嘘**をつく（実際は6言語対応）。
- **原因**: 前回私が入れた指示「**必ず訪問者言語(=session=ja)で回答／会話中も言語を変えない**」が強すぎ、ユーザーの実際の言語を無視して ja に固定した。
- **修正（`functions/src/utils/ai.ts` プロンプト）**:
  - 「**お客様が書いている言語で答える**。英語で来たら英語、日本語なら日本語…。**明示指定（"English please" 等）に必ず従う**」。
  - 「参照情報（ホスピタリティ/RAG/顧客情報）が日本語でも、それは内部資料。**回答言語には影響させない**」。
  - 「**『日本語のみ対応』等とは絶対に言わない**（ja/en/zh/ko/th/vi 対応）」。
  - 「会話中も言語を変えない」の一文を**削除**（これが固定化の元凶）。
  - 返す `language` フィールドは**実際に回答した言語**。

### ② 既定言語が日本語になっている（英語にすべき）【P0】
- **症状**: 何もしないと日本語で開始（ブラウザが日本語ロケールのため）。
- **要件**: 本商品は訪日外国人向け＝**既定は英語**。
- **修正（`client` の言語決定）**: ウィジェットの言語初期値を **`en` 既定**にする（`navigator.language` に引っ張られない）。ユーザーが言語切替/`?lang=` を明示した場合のみそれに従う。→ `session.language` も既定 en。※ `LanguageContext`/`useLanguage` の初期化を実コードで確認して調整。

### ③ すべてエスカレーション扱いになる（挨拶でも escalated）【P0】
- **症状**: "hello" や "test" の**単なる挨拶でも `resolved=false` → エスカレーション**（管理画面バッジ）。
- **原因**: AIが `resolved` を過剰に false にしている（挨拶＝「問題未解決」と解釈）。
- **修正（`ai.ts` プロンプト）**:
  - 「**挨拶・雑談・通常のやり取り・情報提供は `resolved=true`**。既定は true」。
  - 「`resolved=false` は**本当に行き詰まった時だけ**（知識なしで案内も不可／人間対応を明確に希望／案内後も未解決）」を再強調。

### ④ 「お問い合わせフォームを開く」ボタンが即出る（最後の手段のはず）【P0】
- **症状**: 挨拶 "test" の直後からボタンが表示。
- **原因**: ③の `resolved=false` 過剰 → ボタン表示条件（最新AIメッセージが resolved=false）が即成立。
- **修正**:
  - **主因は③**。③を直せばボタンは「本当に解決できない時」だけ出る。
  - 追加ガード（`client/ChatWidgetFirebase.tsx`）: 表示条件を **「最新AI回答が resolved=false **かつ** 会話が1往復以上」** に厳格化（初手では出さない）。※任意だが「最後の手段」感を担保。

## 検証（dev・英語ブラウザ/日本語ブラウザ両方）
1. **既定英語**: 何もせず開く → 挨拶が**英語**で始まる（②）。
2. **言語追従**: 日本語で書けば日本語、英語で書けば英語。"English please"/"日本語で" に即追従。「日本語のみ」と言わない（①）。
3. **非エスカレーション**: "hello"/"test"/通常Q&A は **escalated にならない**／CONTACTボタンも**出ない**（③④）。
4. **正しいエスカレーション**: 本当に答えられない質問でだけ、謝意＋CONTACTボタン＋`escalated`（③④）。
5. `tsc`（client/functions）＋ `vite build`。

## デプロイ
- `pnpm deploy:functions`（①③）＋ `pnpm deploy:dev`（②④）→ dev再検証 → 本番 `pnpm deploy:hosting`。

## 補足（正直な原因整理）
- ①②は「日本語化バグ」を直そうとした前回修正が**逆方向に振れすぎた**もの。正しくは「固定」ではなく「**ユーザーに追従＋既定英語**」。
- ③④はエスカレーション再定義で `resolved` の粒度が表面化したもの。`resolved` の既定を true に寄せれば解消。
